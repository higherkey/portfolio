THE CHALLENGE

Hobbyist board games like *Catan* or *Terraforming Mars* often lack a unified digital companion that handles the "boring" parts of gaming: keeping score, finding groups, and tracking long-term win rates. The goal was to build a platform that feels like an extension of the physical board game, not a replacement for it.

DEVBLOG

Building for **Architectural Velocity** meant automating everything. I established a pipeline where every commit to `main` triggers a multi-stage deployment:

  • Backend (C#) deploys to Render via Docker containers.
  • Frontend (Angular) deploys to Vercel for edge delivery.
  • Database migrations are handled automatically through Supabase CLI.

The real technical challenge was the real-time lobby system. By using SignalR hubs, I reduced state-sync latency to under 50ms, ensuring that when a player joins a lobby, everyone else sees them instantly.
